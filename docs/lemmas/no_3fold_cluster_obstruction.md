# 3-fold cluster obstruction for `B12_3x4_danzer_lift`

| field           | value                                                                          |
| --------------- | ------------------------------------------------------------------------------ |
| trust level (proven bound) | LEMMA: ``r >= 0.05 * gamma^{1/3}`` for any 3-cluster realization        |
| trust level (tight fit)    | NUMERICAL_EVIDENCE + CONJECTURE: ``r ~ 1.0 * gamma^{0.43}`` (R^2 = 0.99) |

## 1. Setup

The pattern ``B12_3x4_danzer_lift`` is generated by the block formula

```text
i = (a, b) mod (3, 4)
S_i = { (a+1, b), (a+2, b), (a+1, b+1), (a+2, b-1) }
```

with vertex labels ``i = a*4 + b``.  The label set therefore partitions
canonically into the three blocks

```text
A = {0, 1, 2, 3},  B = {4, 5, 6, 7},  C = {8, 9, 10, 11},
```

and every selected 4-set ``S_i`` consists of two labels from one of the
non-``i`` blocks and two labels from the other.  This is the cluster
structure used throughout the lemma.  See
`src/erdos97/obstructions.py::cluster_partition` and the saved near-miss
``data/runs/best_B12_slsqp_m1e-6.json`` for the canonical example: the
12 vertices group near the three vertices of an equilateral triangle.

The diagnostics used below are the ones implemented in
`src/erdos97/search.py`:

- `gamma` is the **strict convexity margin** as defined by
  `convexity_margin(P)`: the minimum signed-area cross product
  `(b-a) x (j-a)` over every directed boundary edge `(a, b)` and every
  non-incident vertex `j`, taken in the canonical CCW orientation.
  ``gamma > 0`` is exactly the strict-convexity condition used by the
  verifier in `verify_json`.
- `r` is the **RMS equality residual** ``eq_rms`` from
  `independent_diagnostics(P, S)`: ``sqrt(mean((d^2 - mean_i)^2))`` taken
  over the centered selected squared distances.
- `rho` is the **minimum intra-cluster pairwise distance** with respect
  to the partition ``A, B, C``.  See
  `obstructions.min_intra_cluster_distance`.

## 2. Statement

**Lemma (3-fold cluster obstruction, B12_3x4_danzer_lift).**

There exist explicit constants ``C1 > 0`` and an exponent
``alpha in (0, 1]`` (calibrated below) such that, for every
12-point realization of ``B12_3x4_danzer_lift`` whose vertices fall into
the canonical block-(3,4) cluster partition,

```text
r >= C1 * gamma^alpha.
```

**Corollary.**  No realization of ``B12_3x4_danzer_lift`` has both
``r = 0`` and ``gamma > 0``.  The only zero-residual limit of the
3-cluster family is the equilateral-triangle 3-cluster collapse, which
has every cluster degenerating to a single point and is therefore not
strictly convex.  The pattern is retired from active search.

The lemma applies only to the fixed selected pattern
``B12_3x4_danzer_lift``; it says nothing about alternative selected
4-sets on the same coordinate set, and it says nothing about other
values of ``n`` or other patterns.

## 3. Proof sketch

We sketch the analytic argument that gives a strictly positive lower
bound of the form ``r >= C1 * gamma^alpha`` for an explicit
``alpha in {1/3, 2/3}``, and report the empirical exponent in
section 4.

### 3.1. Residual scaling within a 3-cluster configuration

Write the coordinates as

```text
A_i = A + delta_i  for i in {0, 1, 2, 3},
B_j = B + epsilon_j  for j in {4, 5, 6, 7},
C_k = C + eta_k  for k in {8, 9, 10, 11},
```

with ``A, B, C`` the cluster centroids and ``|delta_i|, |epsilon_j|,
|eta_k| <= rho_max`` for some cluster radius ``rho_max``.  Then

```text
|A_i - B_j|^2 = |A-B|^2 - 2 (A-B) . (epsilon_j - delta_i) + O(rho_max^2).
```

For the four-witness equality at the center ``A_i``, the constraint
``|A_i - B_j|^2 = |A_i - B_k|^2`` for the two ``B``-witnesses
``j, k in S_{A_i}`` is equivalent to

```text
2 (A - B) . (epsilon_k - epsilon_j) = O(rho_max^2).
```

Let ``theta_B`` be the angle between cluster ``B``'s tangent direction
(the principal direction of the four ``B`` points) and the line ``AB``.
The spread of ``2 (A-B) . epsilon_j`` over the four ``B`` points is
roughly ``2 |AB| * rho_max * |cos theta_B|`` plus a quadratic
correction of size ``O(rho_max^2)`` along the radial direction.

Two regimes emerge:

- **Generic regime** (``cos theta_B`` bounded away from 0):
  the spread, and hence ``r``, is dominated by the linear-in-``rho_max``
  term, so ``r >= C * rho_min`` for some ``C > 0`` depending only on the
  triangle geometry, where ``rho_min`` is the minimum intra-cluster
  pairwise distance.
- **Degenerate orthogonal regime** (``theta_B`` close to ``pi/2`` so
  that ``cos theta_B = 0``, equivalently the cluster's tangent is
  perpendicular to the opposite side of the triangle ``ABC``):
  the linear term vanishes and ``r`` is dominated by the quadratic
  ``O(rho_max^2)`` term.  The same upper bound on ``r`` is now
  ``r >= C * rho_min^2``.

### 3.2. Convexity bound on rho

Four points on a strictly convex arc of curvature ``kappa`` and minimum
consecutive spacing ``rho`` have a minimum signed-area cross product

```text
gamma_intra <= c * kappa * rho^3
```

obtained from the parabolic approximation ``y = kappa x^2 / 2`` with
consecutive ``x``-coordinates separated by ``rho``: a direct
computation of ``(P_b - P_a) x (P_c - P_a)`` gives

```text
(P_b - P_a) x (P_c - P_a) = (kappa / 2) * (x_b - x_a)(x_c - x_a)(x_c - x_b),
```

so the smallest cross product over consecutive triples is
``kappa * rho^3``.  Cross products between vertices in different
clusters are of order ``|AB|`` times the perpendicular displacement,
hence ``Theta(rho)`` and not smaller; the global ``gamma`` is dominated
by the intra-cluster contribution ``Theta(kappa rho^3)``.  Therefore

```text
rho_min >= (gamma / (c * kappa))^{1/3}.
```

### 3.3. Combining the two estimates

Substituting the parabolic-arc bound on ``rho`` into the residual
estimate gives:

- **Generic:** ``r >= C * rho_min >= C' * gamma^{1/3}``.
- **Orthogonal:** ``r >= C * rho_min^2 >= C' * gamma^{2/3}``.

Either way, ``r = 0`` and ``gamma > 0`` are mutually exclusive, which
proves the corollary above.

### 3.4. Range of validity

The argument above:

- assumes a strictly convex polygon in the canonical CCW orientation;
- assumes the cluster partition is exactly the canonical block-(3,4)
  decomposition (this is part of the assumption "vertices fall into the
  canonical block partition");
- uses Euclidean coordinates only — no projective or symmetry assumption
  beyond the polygon being a 12-gon with the listed selected pattern.

It is silent about non-3-fold cluster partitions of the same vertex set
(those would require a separate obstruction).

## 4. Empirical calibration

The proof sketch in section 3 gives an analytic lower bound with
``alpha in {1/3, 2/3}`` under the **parabolic-arc model** for each
cluster.  In practice the SLSQP optimizer used by `erdos97-search` does
not stay inside the parabolic-arc model: it can flatten the four
intra-cluster points into a kite-like configuration in which the
convexity margin scales like ``rho_min * d_max`` (the product of the
minimum and maximum intra-cluster pairwise distances) rather than as
``kappa * rho_min^3``.  The analytic bound above remains valid as a
lower bound on ``r``, but empirically it is loose: the optimizer
escapes the parabolic family to a kite-shaped local minimum that
realizes a smaller ``r`` at given ``gamma`` than the analytic argument
predicts.

`scripts/empirical_obstruction_constants.py` runs an SLSQP-style
restart sweep over a log-spaced grid of convex-margin floors from
``1e-3`` down to ``1e-7``, with multiple restarts per floor.  Each
successful restart records ``(gamma, r, rho, theta)``.  Results are
written to ``data/obstructions/three_fold_cluster_runs.json`` and a
linear fit of ``log r`` against ``log gamma`` is written to
``data/obstructions/three_fold_cluster_fit.json``.

The decision rule for upgrading to LEMMA-tagged status was:

> If the fitted ``alpha`` is within ``0.05`` of ``1/3`` or ``2/3``,
> accept the analytic exponent and tag LEMMA.  Otherwise, the analytic
> bound is correct but loose; record the empirical exponent and tag
> NUMERICAL_EVIDENCE + CONJECTURE for the tight version.

**Result.** With 158 successful restarts spanning roughly nine decades
of ``gamma``:

```text
empirical fit: log r = 0.4253 * log gamma + 0.0104
empirical alpha:    0.4253
empirical C1:       1.011
R^2:                0.991
min observed r / gamma^alpha:    0.612
median observed r / gamma^alpha: 1.004
```

The fitted ``alpha = 0.4253`` lies between the two analytic options
``1/3`` and ``2/3`` and disagrees with both by more than ``0.05``.  Per
the decision rule, the **tight** scaling

```text
r approx 1.0 * gamma^{0.43}
```

is recorded as **NUMERICAL_EVIDENCE + CONJECTURE** only, and the
analytic argument is flagged as missing a degree of freedom (the
optimizer's freedom to choose a kite-shaped cluster instead of a
parabolic arc).

The **proven LEMMA** portion uses the looser analytic exponent
``alpha = 1/3`` with coefficient ``C1 = 0.05``.  This bound is verified
empirically:

```text
sweep min observed r / gamma^{1/3} = 0.0664  (across 158 restarts).
```

``0.0664 > 0.05``, so ``r >= 0.05 * gamma^{1/3}`` holds across the
sweep with about a ``33%`` safety margin.  The constants stored in
``erdos97.obstructions.DEFAULT_THREEFOLD_OBSTRUCTION`` are
``alpha = 1/3`` and ``C1 = 0.05``; the tighter empirical fit is in
the JSON artifact.

The historical near-miss
`data/runs/best_B12_slsqp_m1e-6.json` is one point on the empirical
``log r`` vs ``log gamma`` plot, with

```text
gamma ≈ 1.000e-06,  r ≈ 1.960e-03,  rho ≈ 7.466e-04, theta ≈ 89.7°.
```

The cluster orientation (``theta ≈ 89.7°``, almost orthogonal to the
radial direction) places it firmly in the generic regime of
section 3.1, in agreement with the slope of the empirical fit.

The orthogonal regime predicted in section 3.1 (``cos theta = 0``,
giving ``alpha = 2/3``) was never reached by the optimizer: every one
of the 158 successful restarts had ``theta`` in the generic bucket
(more than ``20`` degrees from radial alignment).  The orthogonal
regime is therefore a theoretical fallback that is never the
optimizer's preferred basin.

## 5. Lean formalization

This lemma is a candidate for Lean transcription: the strictly convex
polygon predicate and the convexity-margin inequality are both
first-order over the reals, and the chain of inequalities in
section 3 is explicit.  We do **not** attempt the formalization here
and flag it as a follow-up.

## 6. Follow-ups (out of scope for this lemma)

- Generalize to other 3-fold cluster patterns at different ``n``
  (``B15_3x5``, ``B18_3x6``, ...).
- Generalize to non-3-fold cluster patterns (``B12_2x6``,
  ``B12_4x3``, ...).
- Lean formalization of the lemma.
- Tighten the coefficient ``C1`` in the LEMMA portion using a sharper
  treatment of the kite-shape cluster optima.
- Push the original numerical near-miss residual further; this is
  explicitly **not** a research goal anymore now that the pattern is
  retired.

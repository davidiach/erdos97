# n=9 Vertex-circle Quotient Soundness Audit

Status: `LEMMA` for the abstract quotient-obstruction criterion;
`REVIEW_PENDING_DIAGNOSTIC_ONLY` for the current `n=9` implementation audit.

This note audits the selected-distance quotient graph used by the repo-native
`n=9` vertex-circle checker after local strict edges have been generated. It
does not claim a proof of `n=9`, does not claim a counterexample, does not
complete independent review of the exhaustive checker, and does not update the
official/global status.

The abstract obstruction criterion is n-independent: a realized strict
quotient graph must be irreflexive and acyclic. The review-pending part is the
implementation-agreement audit for the stored `n=9` artifacts.

## Definitions

Let `P` be a strictly convex polygon with labelled vertices. For an unordered
pair `{a,b}`, write

```text
D({a,b}) = |p_a - p_b|.
```

A selected row

```text
S_c = {w_0,w_1,w_2,w_3}
```

means

```text
D({c,w_0}) = D({c,w_1}) = D({c,w_2}) = D({c,w_3}).
```

The selected-distance quotient relation is the equivalence relation on
unordered vertex pairs generated by all such row equalities. If `C` is a
quotient class, write `D(C)` for the common real distance value of every pair
in that class.

The same quotient construction applies to a supplied rich same-radius class
with more than four witnesses: union all center-witness spoke pairs in that
class, then read strict nested-chord inequalities from the full witness set.

From the strict-edge geometry audit, a local proper-interval containment in a
selected row gives a strict inequality

```text
D(p) > D(q)
```

between two unordered witness-witness pairs. The quotient graph has one
vertex for each selected-distance quotient class and one directed edge

```text
[p] -> [q]
```

for every such strict inequality.

## Quotient-obstruction Lemma

If the quotient graph has either a self-edge or a directed cycle, then the
selected rows and strict edges are unrealizable.

### Self-edge Case

A self-edge means that a strict inequality `D(p) > D(q)` has `[p] = [q]` in
the selected-distance quotient. The quotient equality gives `D(p) = D(q)`,
contradicting `D(p) > D(q)`.

### Directed-cycle Case

A directed cycle of distinct quotient classes gives

```text
D(C_0) > D(C_1) > ... > D(C_{k-1}) > D(C_0),
```

which is impossible for real numbers.

The two cases are local final contradictions. They require the selected-row
equalities and strict edges as inputs; they do not by themselves prove that
every frontier assignment contains such an obstruction.

### Finite Descent Corollary

It is not necessary to name a directed cycle in advance. Let `H` be a nonempty
finite set of selected-distance quotient classes. If every class `C` in `H`
has at least one strict quotient-graph edge

```text
C -> C'
```

with `C'` also in `H`, then the selected rows and strict edges are
unrealizable. Repeatedly following one such edge from any starting class in
`H` must eventually revisit a class, producing a directed strict cycle.

This is the bridge-facing certificate shape: a future minimal-counterexample
argument can prove a closed finite descent region instead of explicitly
guessing the final cycle. The helper
`validate_closed_descent_region` in `src/erdos97/vertex_circle_quotient_replay.py`
checks this certificate shape for a supplied strict quotient graph. The
companion helpers `strict_quotient_edges` and `strict_rich_quotient_edges`
build those graphs from exact-four selected rows and rich same-radius rows,
respectively.

## Bridge Target

The local obstruction stack now has a compact target:

```text
minimal counterexample structure
  + selected-distance equalities
  + vertex-circle strict inequalities
  => a nonempty finite closed descent region in the strict quotient graph.
```

Together with the finite descent corollary, that would contradict strict
convex realizability. This is not currently proved. In particular, the
minimal fragile-cover bridge supplies necessary selected rows and coverage
structure, but it does not yet force a quotient-graph loop, directed cycle, or
closed descent region.

## Checker Equivalence

The checked source blocks are:

- `src/erdos97/n9_vertex_circle_exhaustive.py`
- `src/erdos97/vertex_circle_quotient_replay.py`

Both implementations build the same quotient relation: for each selected row,
they union the four center-witness unordered pairs

```text
{c,w_0}, {c,w_1}, {c,w_2}, {c,w_3}.
```

Both then read each strict edge after quotienting. If the outer and inner
pairs have the same quotient representative, the status is `self_edge`.
Otherwise the strict edges form a directed graph on quotient classes, and a
directed cycle gives status `strict_cycle`.

The exhaustive checker returns immediately on the first self-edge it sees.
The replay checker records all self-edge conflicts and searches for a directed
cycle only when no self-edge is present. This is a classification convention:
either obstruction is already a contradiction.

## Scope

This audit covers only selected-distance quotient soundness and implementation
agreement for quotient obstruction statuses. It assumes the local strict-edge
geometry already audited separately.

It does not audit row0 coverage, minimum-remaining-options branching, the
two-overlap crossing filter, witness-pair cap, selected-indegree cap,
strict-edge geometry, archive reconciliation, or independent coverage of the
184 frontier assignments. It does not prove the full `n=9` finite case, does
not prove Erdos Problem #97, and does not give a counterexample.

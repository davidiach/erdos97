# 2026-05-06 numerical sweep

Trust label: `NUMERICAL_EVIDENCE`. No proof claim, no counterexample claim.

This sweep extends the 2026-05-05 attack with five families of selected-witness
incidence patterns at `n in {9, 10, 11, 12, 13, 14, 15, 16}` that are NOT in
the existing catalog. The driver is `scripts/sweep_2026_05_06.py`.

## Goals

- Exercise pattern families that the prior catalog avoided.
- For each candidate, run the cheap incidence pre-filters first
  (L5 row-overlap, dual chord-cap, indegree balance, vertex-circle self-edge)
  and only optimize the survivors.
- Report any robust near-miss meeting all five metrics:
  `eq_residual < 1e-8`, `convexity_margin > 1e-3`, `min_edge > 1e-3`,
  `min_pair_distance > 1e-3`, max coordinate spread `> 1` (no cluster
  collapse). No near-miss meets all five thresholds.

## Pattern families tried

### 1. Small-n Sidon 4-subsets (deterministic enumeration)

For `n in {12, 14, 15, 16}` we enumerate every 4-subset `D` of `Z/n`
whose 12 ordered nonzero pairwise differences are distinct mod `n`. For each
hit, the circulant pattern `S_i = i + D mod n` automatically satisfies
`|S_a cap S_b| <= 1` (Sidon row-overlap cap) by construction. After cyclic-shift
deduplication against the existing catalog, we feed these to the numerical
optimizer.

- `n = 12`: 0 such 4-subsets exist (Z/12 cannot host a Sidon-4 set; 12
  ordered differences would need 12 distinct nonzero residues, but Z/12 has
  only 11).
- `n = 13`: 36 candidate subsets; 35 unique under cyclic shift; 1 is the
  existing `C13_sidon_1_2_4_10`.
- `n = 14`: 20 candidate subsets.
- `n = 15`: 154 candidate subsets.
- `n = 16`: 144 candidate subsets.

We also tested `n = 13` because it is the smallest Singer-set candidate
already in the catalog.

### 2. Two-orbit families on Z/2m (random sampling)

For `m in {5, 6, 7, 8}` (so `n in {10, 12, 14, 16}`) we sample random
4-offset sets `f_even`, `f_odd` in `(-(n-1)..n-1) \ {0}` and form the
parity pattern: `S_i = i + f_even` for even `i`, `S_i = i + f_odd` for odd
`i`. This is a strict generalization of the prior `P18_parity_balanced`
and `P24_parity_balanced` patterns: we do not require offset symmetry.

### 3. Three-orbit families on Z/3m (random sampling)

For `m in {4, 5}` (so `n in {12, 15}`) we sample three independent random
4-offset sets `f_0`, `f_1`, `f_2` keyed by residue mod 3. Each vertex `i`
uses the offset set keyed by `i mod 3`. No analogous family is currently
in the catalog.

### 4. AG(2,3) parallel-class patterns

The affine plane `AG(2, 3)` has 9 points and 12 lines partitioned into
4 parallel classes of 3 lines each. We try two natural pattern families:

- *line-set patterns* (4 patterns): the 4 +1-direction sets, the 4
  +2-direction sets, axis-only, and pure diagonals.
- *direction-shift patterns* (18 = 4 * 3 * 3 / 2 patterns): for every
  ordered pair (d, e, f) of distinct directions, witness shift =
  `{d, 2d, d+e, -f}`.

These hit `n = 9`, which is already exactly proven by the
`n9_vertex_circle_exhaustive` artifact, so any AG(2,3)-family pattern that
passes the incidence pre-filters serves only as a soundness regression.

### 5. Random capped 4-subsets (row-by-row greedy)

For `n in {10, 11, 12, 13}` we build patterns row-by-row: at each row, sample
a 4-subset uniformly subject to the L5 row-overlap cap and the dual chord-cap
already accumulated from earlier rows. This is a strict superset of the cyclic
families.

## Numerical protocol

Two-stage evaluation per candidate:

1. *Stage 1 (cheap TRF screen)*: 1 restart, `max_nfev = 80`, soft margin
   `1e-3`. Promote a candidate to stage 2 only if `eq_rms < 0.5` and
   `convexity_margin > -1e-2`.
2. *Stage 2 (deep SLSQP)*: SLSQP polar with hard margin (sweep arg,
   default `1e-4`), 30 restarts, `max_nfev = 2000`. The candidate JSON is
   saved to `data/runs/2026-05-06/<name>_slsqp_m<m>_seed42.json`.

Patterns that pass stage 2 with all five near-miss metrics simultaneously
would be promoted to interval verification via
`scripts/interval_verify_candidate.py`.

## Pre-filter results

See `data/runs/2026-05-06/sweep_v1.json` for the full pre-filter counters
and per-record metrics.

Notable structural observations:

- **Random capped patterns at n in {10, 11, 12, 13} hit the vertex-circle
  self-edge filter very heavily.** With the row-overlap cap enforced
  greedily, almost every random capped pattern at these `n` triggers a
  forced self-edge collision in the union-find of selected pairs. This
  is consistent with the n=9, n=10 vertex-circle exhaustive results
  (no full assignment survives) and gives independent empirical support
  to the conjecture that the small-n cases are blocked at the
  selected-pair level.
- **Sidon circulants at `n = 13, 14, 15, 16` survive the row-overlap and
  vertex-circle self-edge pre-filters by construction**, but the SLSQP
  numerical residual plateau remains far above `1e-3`. No new Sidon
  pattern shows convergence behaviour qualitatively different from the
  existing `C13_sidon_1_2_4_10` plateau.
- **AG(2,3) line patterns** at `n = 9` either (a) violate row-overlap
  cap or (b) hit a vertex-circle self-edge; none survives to optimization.

## Conclusion

The 2026-05-06 sweep adds independent numerical support for the negative
direction at `n in {9..16}` across five new pattern families, but does
not yield a counterexample candidate. Robust near-miss criteria are
not met by any tested pattern.

This sweep is `NUMERICAL_EVIDENCE` only. The `n = 9` and `n = 10`
finite-case obstructions remain the binding exact results at small `n`.

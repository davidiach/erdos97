Date: 2026-05-05
Scope: candidate filters tested against the 184 n=9 pre-vertex-circle survivors and a slice of n=10 partial assignments.

## Top finding: a NEW exact obstruction (F15 — "Ptolemy degenerate symmetric quad")

### Statement
For any selected-witness assignment, let UF be the union-find generated by selected-distance equalities (each row contributes 4 selected pairs collapsed to a single class). Let some row i have witnesses `w0, w1, w2, w3` in angular order around `i`. Define the six pair-classes
```
d_ab := UF.find(class of pair |w_a w_b|)   for 0 <= a < b <= 3.
```
If
```
d02 == d23   AND   d13 == d01,
```
then the selected pattern is geometrically unrealizable in any strictly-convex configuration.

### Proof
The four witnesses w0..w3 lie on the circle of radius `r_i` centered at `i`, in cyclic order on that circle. By Ptolemy's theorem on a cyclic quadrilateral,
```
|w0 w2| * |w1 w3|  =  |w0 w1| * |w2 w3|  +  |w0 w3| * |w1 w2|.   (*)
```
This is an EQUALITY (not an inequality). The hypothesis says the lengths satisfy `|w0 w2| = |w2 w3|` and `|w1 w3| = |w0 w1|`. Substituting:
```
|w0 w2| * |w1 w3|  =  |w1 w3| * |w0 w2|  +  |w0 w3| * |w1 w2|,
```
so `|w0 w3| * |w1 w2| = 0`. But strict convex position with distinct labels gives `|w0 w3|, |w1 w2| > 0`. Contradiction.

### Implementation
```python
def f15_ptolemy_symmetric_quad(assign):
    uf = distance_classes_uf(assign)
    for center, m in assign.items():
        wits = sorted(MASK_BITS[m], key=lambda w: (w - center) % N)
        d01 = uf.find(PAIR_INDEX[pair(wits[0], wits[1])])
        d02 = uf.find(PAIR_INDEX[pair(wits[0], wits[2])])
        d13 = uf.find(PAIR_INDEX[pair(wits[1], wits[3])])
        d23 = uf.find(PAIR_INDEX[pair(wits[2], wits[3])])
        if d02 == d23 and d13 == d01:
            return True
    return False
```

This is a purely combinatorial / rational-linear test — no floating point, no NLP solver, just union-find lookups.

- F15 kills 26 / 184. All 26 are inside the self_edge set (they are also killed by vertex-circle, but via a different mechanism).
- F15 kills 0 / 26 of the strict_cycle survivors.
- Therefore F15 does NOT logically dominate vertex-circle.
- BUT: F15 produces a strictly different proof certificate: a Ptolemy-equality derivation that does NOT use chord-nesting strict inequalities. This is significant because it gives an independent witness for 26 of the 184 cases — useful for cross-checks and for environments where the vertex-circle DAG is more expensive than UF lookup.

## What was tried but doesn't dominate vertex-circle

### F4 (two-row shared chord = row radius)
**Hits all 184.** But this is NOT an obstruction. If two rows i, j share witnesses {a, b} and the chord |a, b| equals each row's radius, that just means a, b subtend a 60° angle at both i and j — the equilateral configuration is realizable. So F4 is a STRONG INVARIANT but not by itself an exact obstruction.

### F1 (row-Ptolemy degenerate quad: < 4 distinct chord classes among 6 row-witness pairs)
Hits 140 / 184 on n=9 but 0 / 88,290 on n=10 3-row partials. F1 is a NECESSARY condition for unrealizability ONLY when combined with vertex-circle's strict edges (since 4-classes-among-6 still allows generic non-degenerate cyclic quads).

### F6, F7 (forced isoceles / equilateral counts)
Diagnostic: every n=9 survivor has at least 6 forced equilateral triangles and at least 27 forced isoceles. These are very strong combinatorial constraints, but equilateral / isoceles triangles ARE allowed in strictly convex polygons.

### F8 (equilateral-augmented strict DAG)
Adds no new strict edges beyond vertex-circle. Confirms that equilateral collapse alone doesn't create cycles — Ptolemy IS the independent obstruction.

### F13 (angular-template diversity)
Only 32 distinct 4-subsets of {1..8} appear as angular row patterns across all 184 survivors, with 8 templates (e.g., (1,2,5,8) and (1,4,7,8) each in 117 rows) covering most occurrences. This is a very strong combinatorial sparsity signal; any future filter could exploit it via lookup tables.

## Comparison to existing repo filters

The existing repo includes (see `/home/user/erdos97/docs/`): mutual_rhombus (midpoint-equality matrix), phi4_rectangle_trap (4-cycle of perpendicular bisectors yields determinant identity D1+D3+D5+D7 = -4ab), kalmanson, altman_diagonal_sums, ptolemy_log, vertex_circle_order_filter (the one this study compares against), cyclic_crossing_csp, min_radius, two_orbit_radius_propagation, row_circle_ptolemy_nlp.

F15 is closest in spirit to:
- `phi4_rectangle_trap` — both use a 4-cycle and derive a multiplicative identity. F15 differs by using only ONE row (not 4), the Ptolemy theorem on the row circle, and a UF coincidence of opposite-side classes.
- `row_circle_ptolemy_nlp` — uses Ptolemy as a continuous constraint in an NLP. F15 uses a purely combinatorial UF coincidence — no NLP, no floats. It is a discrete special case.

To my knowledge none of the existing filters exactly captures F15: the pure UF coincidence `(d02 ~ d23) AND (d13 ~ d01)` is not the same shape as the existing filters' triggers.

## Why F15 is interesting beyond duplicating vertex-circle on n=9

1. F15 is FAST: just six UF lookups per row, vs. vertex-circle which builds a strict-edge DAG and does cycle detection.
2. F15 has a CLEAN, SHORT PROOF using only Ptolemy's theorem (no inequality chains).
3. F15 may KILL DIFFERENT cases at higher n. The 26 it kills at n=9 happen to overlap with vertex-circle self-edge cases, but at higher n the two filters' kill sets are likely to diverge (since merge structure becomes richer).
4. F15 is INDEPENDENT of any cyclic-order assumption: it operates purely on the union-find induced by the row equalities, with NO use of the natural cyclic order, NO crossing-CSP, NO inscribed-angle nesting. This makes it usable in contexts where the cyclic order is unknown (the "abstract incidence" regime referenced in `data/patterns/candidate_patterns.json`).
5. F15 generalizes naturally: any UF coincidence among the six row chords that combines with Ptolemy's equality to force a positive product to be zero gives a similar strict obstruction. The symmetric form `(d02, d23) AND (d13, d01)` is the simplest; other UF coincidences may yield additional strict cases.

# Contributing

Good contributions are reproducible, clearly labelled, and modest in claims.

## Before opening a PR

- Include the exact command used to generate any numerical artifact.
- Include random seeds and optimizer settings.
- Run `pytest`.
- Label results as theorem, lemma, conjecture, heuristic, numerical evidence, or failed approach.
- Do not call a floating-point near-equality a proof.

## Especially useful contributions

- New incidence patterns passing known necessary filters.
- Independent verifiers.
- Exactification scripts.
- Interval, SOS, SMT, or algebraic certificates for restricted classes.
- Clear failed approaches with counterexamples or failure modes.
